# Recap Reuse Rules

Prefer updating an existing recap when it is clearly the same session or workstream.

## Strong Reuse Signals

- It is the newest recap under `$PWD/docs/recaps/`.
- Its linked artifacts and scope still match the current work.
- The user asked for a continuation or handoff update instead of a separate recap.

## Create A New Recap When

- The user explicitly asks for a separate artifact.
- The work belongs to a different session or independent workstream.
- Reusing the old note would blur two unrelated stories.
